# 개발 스타일 가이드

## 권선현 개발 스타일

### 1. 병렬 처리 선호
- 독립적인 파일들은 **최대한 병렬로 생성**하여 효율성 극대화
- 파일이 겹치지 않으면 병렬, 같은 파일 수정 필요하면 순차, 의존성 있으면 순차

### 2. 단계별 진행 순서
1. 프로젝트 초기화
2. 설정 파일
3. 라이브러리
4. 컴포넌트
5. 페이지
6. 콘텐츠

### 3. 환경변수는 마지막에
- 개발 완료 후 환경변수 설정

### 4. TodoWrite 활용
- 진행 상황을 체계적으로 추적하고 관리

### 5. 빌드 검증 필수
- `npm run build`로 프로덕션 빌드 테스트 완료 후 보고

---

## 코드 스타일 가이드

- TDD 방식 사용
- 함수는 **50줄 이하**로 작성
- 테스트: `npm test`
- 린트: `npm run lint`

---

## 기능 구현 자동 분할 (7개 Task)

| Task | 내용 |
|------|------|
| 1 | Component (메인 컴포넌트) |
| 2 | Styles (CSS) |
| 3 | Tests (테스트) |
| 4 | Types (타입 정의) |
| 5 | Hooks (커스텀 훅) |
| 6 | Integration (라우팅, import) |
| 7 | Config (설정 파일) |

- 각 Task 독립 실행 후 완료 순서대로 자동 머지
- 테스트 통과 시 자동 머지, 실패 시 보고, 충돌 발생 시 중단 후 보고

---

## 서브에이전트 위임 규칙

- 코드 작성 → developer 에이전트
- 코드 완성 후 → reviewer 에이전트 (자동)
- 테스트 실패 → tester 에이전트 (자동)
- 보안 이슈 → security-auditor 에이전트 (자동)

---

## UI 스타일 선호

### 코드 블록 복사 버튼
- 노션 스타일로 구현
- 코드 블록 내부 우측 상단에 아이콘으로 배치
- 호버 시에만 표시
- 복사 완료 시 체크 아이콘으로 변경
- 텍스트 대신 SVG 아이콘 사용
